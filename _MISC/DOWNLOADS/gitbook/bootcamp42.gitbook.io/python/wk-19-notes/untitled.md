# <span class="text-4505230f--DisplayH900-bfb998fa--textContentFamily-49a318e1">D4</span>

<span class="text-4505230f--UIH300-2063425d--textUIFamily-5ebd8e40--text-8ee2c8b2"></span>

<span class="text-4505230f--UIH300-2063425d--textUIFamily-5ebd8e40--text-8ee2c8b2"></span>

<span class="text-4505230f--UIH300-2063425d--textUIFamily-5ebd8e40--text-8ee2c8b2"></span>

<span class="text-4505230f--HeadingH700-04e1a2a3--textContentFamily-49a318e1"><span data-key="3c3654347fe54345a2185a450ea1bac1"><span data-offset-key="3c3654347fe54345a2185a450ea1bac1:0">Overview</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="0d0d5ecf511d4e3c86b02c059102a336"><span data-offset-key="0d0d5ecf511d4e3c86b02c059102a336:0">There is only one way to traverse linear data structures like arrays, linked lists, queues, and stacks. The linear nature of the structure itself forces a particular type of traversal.</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="988374d86994402aa8edc34ba0fe89b3"><span data-offset-key="988374d86994402aa8edc34ba0fe89b3:0">However, with hierarchical structures like trees, there are multiple ways that you can traverse the stored data. There are two primary categories for tree traversals:</span></span></span>

1.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="fb1f33c3ff434640a8539becf14c3de3"><span data-offset-key="fb1f33c3ff434640a8539becf14c3de3:0">Depth-First</span></span></span>

2.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="14b57ecd98334997a284c203815ad5e8"><span data-offset-key="14b57ecd98334997a284c203815ad5e8:0">Breadth-First</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="65414f05459e45f6a8d1adab082a3419"><span data-offset-key="65414f05459e45f6a8d1adab082a3419:0">Furthermore, there are three different types of depth-first traversals:</span></span></span>

1.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="a3fb1e98e58a4e90ad5116a9c61fd56b"><span data-offset-key="a3fb1e98e58a4e90ad5116a9c61fd56b:0">Inorder</span></span></span>

2.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="9ab4490567ce476f914137e089e7a02c"><span data-offset-key="9ab4490567ce476f914137e089e7a02c:0">Preorder</span></span></span>

3.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="120aaaf1ffee4eb9a337955b1329d46f"><span data-offset-key="120aaaf1ffee4eb9a337955b1329d46f:0">Postorder</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="db13b984d677454aaef0c0111b0623f8"><span data-offset-key="db13b984d677454aaef0c0111b0623f8:0">Let's dive deeper into each of the traversal types.</span></span></span>

<span class="text-4505230f--HeadingH700-04e1a2a3--textContentFamily-49a318e1"><span data-key="d50d7b106fd941d1b4e52eacabfbbade"><span data-offset-key="d50d7b106fd941d1b4e52eacabfbbade:0">Follow Along</span></span></span>

<span class="text-4505230f--HeadingH600-23f228db--textContentFamily-49a318e1"><span data-key="49fccc1be1724e788f198ea863983a3e"><span data-offset-key="49fccc1be1724e788f198ea863983a3e:0">Depth-First Inorder Traversal</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="100760a9975e4ff8a1de1047736112f1"><span data-offset-key="100760a9975e4ff8a1de1047736112f1:0">Let's first look at an inorder depth-first traversal of a binary tree. In this traversal, we start at the tree's root node and complete the following steps recursively:</span></span></span>

1.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="d46e4780bd034f4b93c43c9da07addeb"><span data-offset-key="d46e4780bd034f4b93c43c9da07addeb:0">Go to the left subtree</span></span></span>

2.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="87f1c30010fa49ffbe0331364dcce19c"><span data-offset-key="87f1c30010fa49ffbe0331364dcce19c:0">Visit node</span></span></span>

3.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="c173b4725f1c4d5688bf04bf80de8fbb"><span data-offset-key="c173b4725f1c4d5688bf04bf80de8fbb:0">Go to the right subtree</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="75e03ff8de1645809e8ace2332aadcde"><span data-offset-key="75e03ff8de1645809e8ace2332aadcde:0">Notice that we don't actually "visit" a node until we've already gone to the left subtree. The base cases in the recursion are when there is no left or right subtree to visit.</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="08eeff1213fc46499f141460ecfd8b58"><span data-offset-key="08eeff1213fc46499f141460ecfd8b58:0">This traversal type is very similar to an inorder traversal except that the three steps' order is slightly different. 1.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="798c32befbc54f7f8007ab33cf471bb0"><span data-offset-key="798c32befbc54f7f8007ab33cf471bb0:0">Visit the node</span></span></span>

2.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="f8655815c8574379b5779f0ea50e42e8"><span data-offset-key="f8655815c8574379b5779f0ea50e42e8:0">Go to the left subtree</span></span></span>

3.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="30ca5deb9b634ea2963008239edb435c"><span data-offset-key="30ca5deb9b634ea2963008239edb435c:0">Go to the right subtree</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="4965dae7d94d47e6b0da5ce92b5343ea"><span data-offset-key="4965dae7d94d47e6b0da5ce92b5343ea:0">Below is the visualization for how this would type of traversal would look.</span></span></span>

<span class="text-4505230f--HeadingH600-23f228db--textContentFamily-49a318e1"><span data-key="7bc85763345d4aebb50ce6a7ff61150c"><span data-offset-key="7bc85763345d4aebb50ce6a7ff61150c:0">Depth-First Postorder Traversal</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="b58dfbb85c61481cad2134777015d337"><span data-offset-key="b58dfbb85c61481cad2134777015d337:0">This traversal type is very similar to our other traversals except that the three steps' order is slightly different. Notice that in this traversal, we "visit" the node (denoted in the visualization below by the node turning red) after we recurse to the left subtree (we represent the recursive call by turning the node grey in the visualization below) and the right subtree.</span></span></span>

1.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="4507fec5b55a42c69afc3a0e22edb033"><span data-offset-key="4507fec5b55a42c69afc3a0e22edb033:0">Go to the left subtree</span></span></span>

2.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="56ef0445c27f44abb620fdc368d2b551"><span data-offset-key="56ef0445c27f44abb620fdc368d2b551:0">Go to the right subtree</span></span></span>

3.  <span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="6565dee3558645378b9fd6ffbadd24c9"><span data-offset-key="6565dee3558645378b9fd6ffbadd24c9:0">Visit node</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="18ce172bbae744d1876c5615ecacd30c"><span data-offset-key="18ce172bbae744d1876c5615ecacd30c:0">Below is the visualization for how this would type of traversal would look.</span></span></span>

<figure><img src="https://tk-assets.lambdaschool.com/41bc2877-94d4-4103-885b-c396bec4832a_depth-first-postorder-traversal.gif" alt="https://tk-assets.lambdaschool.com/41bc2877-94d4-4103-885b-c396bec4832a_depth-first-postorder-traversal.gif" class="image-52799b3c" /><figcaption><span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1" style="max-width:100%">https://tk-assets.lambdaschool.com/41bc2877-94d4-4103-885b-c396bec4832a_depth-first-postorder-traversal.gif</span></figcaption></figure>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="e896b07c7dd04c8c9ac7a2cfb57150ab"><span data-offset-key="e896b07c7dd04c8c9ac7a2cfb57150ab:0">Here is one possible way to code a depth-first postorder traversal in Python:</span></span></span>

    class TreeNode:     def __init__(self, val=0, left=None, right=None):         self.val = val         self.left = left         self.right = right​def helper(root, res):    if root is None:        return    helper(root.left, res)    helper(root.right, res)    res.append(root.val)​def postorder_traversal(root):    result = []    helper(root, result)    return result

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="10e3c5ef26444040b412fd0fd0dd0df1"><span data-offset-key="10e3c5ef26444040b412fd0fd0dd0df1:0">Notice that the only difference between the code above for postorder traversal and the other examples is that in this version, we append the node's value to the result only after we've already recursed to the left and right subtrees.</span></span></span>

<span class="text-4505230f--HeadingH600-23f228db--textContentFamily-49a318e1"><span data-key="0028e2b6eaa04fd48836c349ed9d8090"><span data-offset-key="0028e2b6eaa04fd48836c349ed9d8090:0">Breadth-First (Level Order) Traversal</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="a49cb8b860874fe686554ff29e306fa6"><span data-offset-key="a49cb8b860874fe686554ff29e306fa6:0">In a breadth-first traversal, we visit all the nodes at the same level (same distance from the root node) before going on to the next level.</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="64ec4bf7cdf94ef49e6e5433a727a4a8"><span data-offset-key="64ec4bf7cdf94ef49e6e5433a727a4a8:0">A breadth-first traversal and a level order traversal are the same things. However, a breadth-first traversal can be done on any hierarchical data structure like trees and graphs. But, a level order traversal refers only to the traversal of a tree. Graphs do not have levels like trees do, so that term would not make sense.</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="a1f0c0f44aac4d6e8804d7be8eab0af0"><span data-offset-key="a1f0c0f44aac4d6e8804d7be8eab0af0:0">A breadth-first traversal is a little different than the depth-first traversals we've gone over. We cannot merely use the recursive call stack to keep track of where we are in the tree. Instead, we must use a queue to keep track of what nodes we should visit. Remember that a queue data structure follows a first-in-first-out (FIFO) access order.</span></span></span>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="17cd005fdf124329b55c520a523775e5"><span data-offset-key="17cd005fdf124329b55c520a523775e5:0">Below is a visualization for a breadth-first traversal.</span></span></span>

<figure><img src="https://tk-assets.lambdaschool.com/671a11b7-acee-4b16-9452-d42f3b69a24e_breadth-first-traversal.gif" alt="https://tk-assets.lambdaschool.com/671a11b7-acee-4b16-9452-d42f3b69a24e_breadth-first-traversal.gif" class="image-52799b3c" /><figcaption><span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1" style="max-width:100%">https://tk-assets.lambdaschool.com/671a11b7-acee-4b16-9452-d42f3b69a24e_breadth-first-traversal.gif</span></figcaption></figure>

<span class="text-4505230f--TextH400-3033861f--textContentFamily-49a318e1"><span data-key="f6133720e3f448db8155235933a4d382"><span data-offset-key="f6133720e3f448db8155235933a4d382:0">Here is one way that you could code a breadth-first (level order) traversal in Python:</span></span></span>

    class TreeNode:     def __init__(self, val=0, left=None, right=None):         self.val = val         self.left = left         self.right = right​def breadth_first_traversal(root):    if root is None:        return []​    result = []    queue = []    queue.append(root)​    while len(queue) != 0:        node = queue.pop(0)        result.append(node.val)​        if node.left is not None:            queue.append(node.left)​        if node.right is not None:            queue.append(node.right)​    return result

<span class="text-4505230f--HeadingH700-04e1a2a3--textContentFamily-49a318e1"><span data-key="2b85d81e31df4f17af39d7ddb1d7f5cb"><span data-offset-key="2b85d81e31df4f17af39d7ddb1d7f5cb:0">Challenge</span></span></span>
